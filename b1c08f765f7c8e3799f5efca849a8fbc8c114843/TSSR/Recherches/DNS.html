DNS

DNS → Domain Name System

Port : UDP 53

Qu'est ce que c'est

Le DNS a été inventé pour faciliter la recherche d'un site donné sur internet. Il permet d'associer un nom compréhensible à une adresse IP. On associe donc une adresse logique, le nom de domaine, à une adresse physique, l'adresse IP.

L'adresse IP et le nom de domaine sont uniques et cela permet donc à un message d'atteindre son destinataire.

 

Fonctionnement

Lorsque l'on saisie une adresse dans le navigateur, c'est un serveur DNS qui traduit cette adresse en adresse IP compréhensible par les ordinateurs et les réseaux. C'est ce qu'on appelle la résolution DNS. Ce temps est d'autant plus faible sur le serveur est performant.

L'architecture logique du DNS est similaire à la structure hiérarchique des noms de domaine. Le DNS d'un niveau hiérarchique donné délègue au sous-niveau le soin de traiter le sous-domaine suivant, jusqu'au dernier niveau qui lui connaît l'adresse IP correspondant au nom de domaine demandé.

 

Il existe plusieurs types de services DNS:

DNS officiel : il fournit un mécanisme de mise à jour que les développeurs utilisent pour gérer les noms publics de DNS. Il répond ensuite aux requêtes DNS, en traduisant des noms de domaine en adresses IP. Le DNS officiel est le principal gestionnaire d'un domaine. Il est chargé de répondre aux serveurs DNS récursifs en leur communiquant l'adresse IP désirée.

DNS récursif : Les clients n'adressent géneralement pas leur requêtes aux services de DNS officiels. Ils se connectent à un résolveur, ou DNS récursif. Il ne possède pas d'enregistrement DNS mais il sert d'intermédiaire pour obtenir les informations pour le client. Si la réference de DNS est mise en cache ou stockée, il répond à la requête DNS en communiquant les informations au client, sinon, il transmet la requête à un ou plusieurs serveurs DNS officiels pour récupérer les informations demandées.

  1. L'utilisateur saisit une adresse dans un navigateur web. La requête est acheminée via internet et est reçue par un résolveur DNS récursif.
  2. Le résolveur interroge alors un serveur de noms racine DNS
  3. Ce dernier répond au résolveur avec l'adresse d'un serveur DNS de domaine de premier niveau sur lequel sont conservées les informations relatives à ses domaines. Ainsi lors d'une recherche, la requête est redirigée.
  4. Le résolveur émet ensuite une requête vers le TLD
  5. Le serveur TLD répond alors avec l'adresse IP du serveur de noms de domaine
  6. Enfin le résolveur récursif envoie une requête au serveur de noms de domaine
  7. Le serveur de noms de domaine envoie ensuite l'adresse IP du site vers le résolveur
  8. Le résolveur DNS répond alors au navigateur web avec l'adresse IP du domaine initialement demandé
  9. Le navigateur envoie une requête HTTP à l'adresse IP
  10. Le serveur situé à cette adresse IP renvoie la page à affcher
