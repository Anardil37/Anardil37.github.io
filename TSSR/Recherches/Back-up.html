Back-up

Sauvegarde complète:

  • Sauvegarde toutes les données du serveur, y compris les apps et l'OS
  • Inclut l'état du système
  • Permet une restauration sur un matériel différent, cependant il est recommandé que le matériel ait la même configuration
  • Sauvegardes plus importantes si beaucoup de données et d'applications
  • Préférable pour restaurer l'ensemble du serveur sur le même serveur ou sur un autre serveur
  • Permet de réinstaller facilement le système d'exploitation et d'utiliser la sauvegarde

Sauvegarde sur un disque → disque dédié pour le stockage des sauvegardes

Sauvegarde sur un volume → par exemple une partition, mais performances du disque réduites

Sauvegarde sur un dossier partagé → sauvegarde à distance

 

La sauvegarde complète d'un Active Directory est une tâche essentielle pour la sécurité et la stabilité d'un réseau. Voici un résumé des points clés :

1. Pourquoi sauvegarder l'Active Directory ?: L'Active Directory est un élément central et critique dans une entreprise. Si l'Active Directory est hors ligne, les conséquences peuvent être désastreuses car les services d'authentification sont inopérants¹. Il est donc indispensable de réaliser une sauvegarde de l'Active Directory pour se protéger contre les cyberattaques et les incidents majeurs.

2. Comment sauvegarder l'Active Directory ? : Pour sauvegarder l'Active Directory, on peut utiliser Windows Server Backup, des outils tiers comme Veeam, Nakivo, HornetSecurity (Altaro), etc., et Azure Backup pour une sauvegarde externalisée dans Azure. La sauvegarde d’un serveur Active Directory doit être effectuée en ligne et doit être effectuée lorsque les services de domaine Active Directory sont installés.

3. Sauvegarde avec Windows Server Backup : Windows Server Backup est capable de sauvegarder les données d'une machine et l'état du système¹. C'est un outil maintenu par Microsoft, et donc pris en charge par Microsoft.

4. Restauration d’un serveur Active Directory : La restauration d’un serveur Active Directory est toujours effectuée hors connexion. Il est important de noter que la restauration de l’état du système doit se produire dans un ordre spécifique.

 

1. Sauvegarde en ligne : La sauvegarde d'un serveur Active Directory doit être effectuée en ligne et doit être effectuée lorsque les services de domaine Active Directory sont installés. Les services de domaine Active Directory reposent sur une base de données spéciale et exportent un ensemble de fonctions de sauvegarde qui fournissent l'interface de sauvegarde par programmation.

2. Sauvegardes incrémentielles : La sauvegarde ne prend pas en charge les sauvegardes incrémentielles lors de l'utilisation de l'utilitaire de Sauvegarde Windows intégré.

3. Restauration hors ligne : La restauration d'un serveur Active Directory est toujours effectuée hors connexion.

4. Ordre de restauration : La restauration de l’état du système doit se produire dans l’ordre suivant :
   - Restaurez les fichiers de démarrage.
   - Restaurez SysVol, serveur de certificats, base de données de cluster et base de données d’inscription de classe COM+, le cas échéant.
   - Restaurez le serveur Active Directory.
   - Restaurez le Registre.

5. Recommandations pour la sauvegarde d’Active Directory : Il est recommandé de sauvegarder au moins deux contrôleurs de domaine, au moins une fois par jour.

6. Utilisation conjointe d'outils de sauvegarde : Il peut être intéressant d'utiliser conjointement Windows Server Backup et des outils tiers de chez Veeam, Nakivo, HornetSecurity (Altaro), etc., ainsi qu'Azure Backup pour une sauvegarde externalisée dans Azure.
