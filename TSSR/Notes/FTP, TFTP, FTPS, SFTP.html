FTP, TFTP, FTPS, SFTP

FTP (File Transfer Protocol) :
  • Signification : FTP est un protocole de transfert de fichiers qui permet d'envoyer et de recevoir des fichiers entre un client et un serveur.
  •  Mode de fonctionnement : Il fonctionne en mode client/serveur, avec un canal de contrôle pour l'authentification et les commandes FTP, et un canal de données pour le transfert des fichiers.
  • Inconvénient : Le principal inconvénient est qu'il n'est pas sécurisé. Les informations telles que les identifiants et les commandes transitent en clair sur le réseau.

 

FTPS (FTP sécurisé) :
  • Signification : FTPS est une version sécurisée de FTP.
  • Sécurité : Il utilise le protocole SSL/TLS pour chiffrer les données et sécuriser les échanges.
  • Ports : Il utilise plusieurs ports (21 pour le contrôle et un port défini pour les données).

 

SFTP (SSH File Transfer Protocol) :
  • Signification : SFTP est un protocole basé sur SSH qui permet le transfert sécurisé de fichiers.
  • Sécurité : Il offre un niveau élevé de sécurité grâce au chiffrement et à l'authentification basée sur clés.
  • Port : Il utilise un seul port (généralement 22) pour toutes les communications.

 

TFTP (Trivial File Transfer Protocol) :
  • Signification : TFTP est un protocole simple de transfert de fichiers, mais il ne permet pas de lister, supprimer, renommer ou modifier les répertoires.
  • Gestion : Contrairement à FTP, il ne possède pas de fonctionnalités de gestion et d'édition de fichiers.

 

En résumé, FTP est le plus ancien et le moins sécurisé, FTPS ajoute une couche de sécurité avec SSL/TLS, SFTP est basé sur SSH et très sécurisé, tandis que TFTP est simple mais limité en fonctionnalité.
