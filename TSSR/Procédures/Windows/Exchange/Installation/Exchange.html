Exchange

Désactiver les mises à jour automatiques avec sconfig

 

Installer netframework 4.8 ainsi que vcredistx64.

Installer ucmaruntimesetup, puis redémarrer.

 

 

Ouvrir PowerShell en administrateur et taper la commande : 

Install-WindowsFeature NET-Framework-45-Features, RPC-over-HTTP-proxy, RSAT-Clustering, RSAT-Clustering-CmdInterface, RSAT-Clustering-Mgmt, RSAT-Clustering-PowerShell, Web-Mgmt-Console, WAS-Process-Model, Web-Asp-Net45, Web-Basic-Auth, Web-Client-Auth, Web-Digest-Auth, Web-Dir-Browsing, Web-Dyn-Compression, Web-Http-Errors, Web-Http-Logging, Web-Http-Redirect, Web-Http-Tracing, Web-ISAPI-Ext, Web-ISAPI-Filter, Web-Lgcy-Mgmt-Console, Web-Metabase, Web-Mgmt-Console, Web-Mgmt-Service, Web-Net-Ext45, Web-Request-Monitor, Web-Server, Web-Stat-Compression, Web-Static-Content, Web-Windows-Auth, Web-WMI, Windows-Identity-Foundation, RSAT-ADDS

 

Installer le module de réécriture et redémarrer.

 

 

Monter l'iso d'Exchange.

Double clic sur Setup.exe se trouvant sur le disque.

 

Choisir de ne pas vérifier les mises à jour.

Ne pas utiliser les paramètres recommandés.

Sélectionner les rôles « Rôle de boîte au lettre » & « Installer automatiquement les rôles et les fonctionnalité Windows Server requis pour Exchanger Server »

Ne pas activer la protection anti programmes malveillants.

Si il n'y a aucun message d'erreur, l'installation peut être effectuée.

 

https://@IP-EXCHANGE/ecp pour accéder à l’interface de gestion exchange.

https://@IP-EXCHANGE/owa pour accéder à l’interface utilisateur.
